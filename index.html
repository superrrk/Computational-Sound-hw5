    Lab 5 Blog

    Automated Composition with Pitch Set Theory

    Project Idea

    For this lab, I built a small browser instrument that composes from pitch set theory operations. The project starts with a pitch class set, then repeatedly applies transpose, inverse, or retrograde to generate a longer sequence. I liked this approach because the rules are simple enough to understand in code, but the musical results still change a lot depending on the starting set and operation mix.

    I wanted the project to feel less like pressing a random button and more like shaping a small compositional system. The controls let the listener change the initial set, the number of transformation steps, the rhythm feel, the tempo, and the probability balance between operations. Those choices do not write a melody directly, but they influence the behavior of the generator in ways that are easy to hear and compare.

    Coding the Transformations

    Coding the transformations made the theory feel more concrete. Transpose was the most direct: add an interval and use modulo 12 so every note stays inside pitch class space. Inversion was more interesting because it changed the contour of the set without introducing any new pitch class names. Retrograde was technically just reversing an array, but musically it made me notice how much order matters even when the same pitch classes are present.

    The modulo 12 part became one of the most important details in the implementation. Without it, transposing by larger intervals could push pitch classes outside the expected range and make the rest of the program harder to reason about. Keeping everything wrapped into 0 through 11 meant that each operation could stay small and predictable, while the longer sequence still became complex over time.

    Operation Mixes

    The operation mix dropdown changes the probability of each transformation, which changes the personality of the generated piece. Balanced gives transpose, inverse, and retrograde a fairly even chance, so the result moves between interval shifts, contour changes, and reversals. Transpose focus favors moving the set up by random intervals, which tends to make the composition feel more harmonically mobile while keeping the shape of the original set recognizable.

    Inverse focus makes the contour flip more often, so the generated material feels more angular and reflective. Retrograde focus keeps returning to reversed versions of the current set, which makes order and direction more noticeable. Restless gives all three operations strong weights, so the system changes direction more often and produces a less predictable sequence. Adding these mixes made the interface feel more musical because I was not only generating notes, I was choosing a behavior for the algorithm.

    This also made me think about probability as a musical parameter. The operation mix does not guarantee that a specific operation will happen at a specific time; it changes the tendency of the system. That feels closer to composing with constraints than composing with exact instructions. I can ask the program to lean toward inversion or transposition, then listen to how that tendency plays out over the generated sequence.

    Making the Process Visible

    The hardest part was not the pitch math; it was making the generation process feel inspectable. I added a step-by-step notes panel because a plain list of numbers did not explain how the composition was built. Each step now shows the input set, the operation chosen, the resulting set, and how many notes have accumulated for playback. That helped turn the project from a black box into something closer to a musical sketchbook.

    The visualization also helped me debug the project. If the sound seemed confusing, I could look at the timeline and the process cards to see whether the issue came from the pitch set transformations, the rhythm pattern, or the playback scheduling. This was especially helpful after adding highlighting, because I could check that the note being heard matched the note being shown on screen.

    Scheduling Sound

    WebAudio also forced me to think about time differently. The notes are not played one at a time with delays; they are scheduled on the audio clock with oscillator start and stop times. I added a short gain envelope to each note so the sound has a clearer attack and release. Once the sound was scheduled, I had to separately coordinate the visual highlighting so the active pitch block and process step lined up with what the listener was hearing.

    I later added rhythm feels because the first version played every note with the same spacing, which made the melody sound too mechanical. The steady pulse keeps the rhythm even, sparse phrases leave more breathing room, syncopated rhythm places shorter notes against longer gaps, and the rolling pattern creates a more continuous forward motion. Accent marks in the timeline show which notes receive a slightly stronger gain envelope during playback.

    The rhythm system changed the piece more than I expected. When every note had the same duration, the output demonstrated the algorithm but did not always feel like a phrase. Adding gaps, accents, and longer sustained notes made the same pitch data feel more intentional. It showed me that automated composition is not only about choosing pitches; timing is a big part of whether the result feels musical.

    Seeded Randomness

    I also added seeded generation because random composition can be frustrating if a good result disappears forever. The seed makes the randomness repeatable, so the same settings can recreate the same sequence. The curated seed presets are examples I chose to demonstrate different personalities quickly: Glass Canon uses a rolling transpose-heavy sound, Mirror Pulse emphasizes inversion and syncopation, and Restless Sparks uses a faster, less predictable mix. That changed how I thought about the interface: the app is not only a generator, but a way to search through possible compositions and return to the ones that work.

    The presets also make the project easier to explore. A listener can still enter their own pitch set and experiment with the algorithm, but the curated examples provide quick starting points that show the system working in different modes. This felt important because randomness can create interesting surprises, but it can also make it harder to return to a musical result unless the settings are easy to save and repeat.

    Takeaway

    My biggest takeaway is that automated composition is a negotiation between rules and presentation. The pitch set operations create the raw musical material, but the rhythm, octave mapping, playback envelope, and visual feedback all shape whether the result feels intentional. Programming those parts myself made the musical system feel less abstract and much more like an instrument.

    If I continued developing this, I would add more timbral variation and maybe a way to export the generated sequence. Still, this version helped me understand the connection between a theoretical pitch operation and an actual listening experience. The most interesting part was seeing how small functions, like transpose or retrograde, could become the foundation for a full interactive composition tool.
